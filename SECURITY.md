# セキュリティポリシー（SECURITY）

## 脆弱性の報告

脆弱性と思われる問題を見つけた場合は、**公開 Issue ではなく**、GitHub の「Report a vulnerability（Private vulnerability reporting / Security Advisories）」から報告してください。

上記が利用できない場合の連絡先（メンテナ）:

- 朝比奈 吾郎（あさひな ごろう）: `asahina.goro@gmail.com`

報告に含めてほしい情報:
- 影響範囲（どのモジュール/スクリプト/設定か）
- 再現手順（可能なら最小構成）
- 想定される影響（情報漏えい、権限昇格、DoS、コスト増など）
- 回避策（もしあれば）

## 対応方針（目安）

- 受領後、状況を確認し、必要に応じて追加情報を依頼します。
- 修正が用意でき次第、リリースノート/CHANGELOG に反映します（調整のうえ公開）。

## 注意（IaC/運用スクリプト特有）

このリポジトリは AWS 上に実リソースを作成するため、設定ミスが **セキュリティ露出**や **課金増**に直結します。特に次を重視します。

- 既定で不要な公開をしない（ALB/WAF/DNS/認証の前提を明確化）
- 秘密情報の平文コミット禁止（tfvars/state/log/evidence）
- 破壊的変更は明示し、ロールバック手順を用意する
